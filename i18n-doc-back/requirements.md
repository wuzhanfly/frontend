# 需求文档：项目国际化

## 任务概述
根据 iflow 文件夹下的组件清单，按照模块顺序(二十六字母正序)，进行国际化处理。

## 结构化要点

### 1. 核心要求
- 按模块字母顺序处理国际化内容
- 不影响原有国际化内容和功能
- 修改须集中批量处理，禁止边改边加
- 严格按照原有国际化格式

### 2. 输出要求
- 完整的国际化项目
- iflow文件夹下以各模块名命名的各模块国际化进度文档
- 每完成一个组件更新一次状态
- 已完成的模块，文档重命名为 模块名-finalized

### 3. 优先级规则
- 硬编码优先处理
- 检查国际化 key 是否存在资源库
- 检查每个 key 的中/英资源是否齐全

### 4. 状态代码定义
- detected: 1 - 检测到需要处理的模块
- processing: 2 - 正在处理
- completed: 3 - 处理完成
- verified: 4 - 已验证
- finalized: 5 - 最终确认

## 范围限制
- 范围: ALL

## 潜在风险
1. 修改现有 key 导致回归
2. 漏掉深层嵌套硬编码
3. 多模块 key 冲突

## 优化建议
1. 使用 schema 校验资源文件
2. 自动扫描硬编码并生成报告
3. PR 流程加入自动 i18n 检查

## 依赖项
1. 组件清单完整性
2. 资源库访问权限
3. CI 校验环境
